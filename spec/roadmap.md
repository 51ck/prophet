# Roadmap

Product milestones only. No engineering tickets, no stack choices.

## Phase 0 — Spec

Pure-idea DOX pack in `spec/`: vision, session, deck/ritual, agent, Telegram UX principles, economics intent, this roadmap.

**Done when:** idea docs describe the product without implementation detail; open UX/econ questions stay labeled open.

## Phase 1 — Ritual core

One deck fully described. Honest shuffle / draw / open. Bounded session arc in chat (intake → lock → ritual → close). Mid-ritual Q&A allowed without breaking card state.

**Done when:** a seeker can complete a real reading whose cards come only from ritual.

## Phase 2 — Prophet voice

Multiple decks. Preference / settings and prophet choice. Richer mid-ritual dialogue and deck-appropriate voice.

**Done when:** deck selection feels intentional and readings differ honestly by deck, not by invented outcomes.

## Phase 3 — Telegram fluency

DM polish. Group summon rules and privacy model. Ceremonial reveal UX (resolve open questions in [telegram-ux.md](telegram-ux.md) as decisions land).

**Done when:** DM readings feel fluid; group summon has an explicit, documented etiquette.

## Phase 4 — Stewardship

Usage limits, abuse and cost control. Prepare for paid access without changing authenticity rules.

**Done when:** runaway cost/abuse paths have product answers; free unlimited drift is not the default forever.

## Phase 5 — Economics

Monetization experiments — likely Telegram Stars or equivalent — per [economics.md](economics.md). Pay for access/capacity, never for nicer cards.

**Done when:** a sustainable access model exists that seekers understand and that leaves ritual honesty intact.

## Ordering rule

Do not pull Phase 5 ahead of proven ritual + usable Telegram feel. Stewardship may overlap late Phase 3 / early Phase 5, but authenticity never waits on payment.
